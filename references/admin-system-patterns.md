# 日常系统管理常见模块模式

当原型包含“系统管理 / 基础管理 / 权限管理”类模块时，遵循下列通用模式。这些模块与业务页面不同：通常以管理、配置、审计为主，数据量中等，结构相似度高，适合用统一的交互约定保持原型一致。

## RBAC 三件套：用户、角色、菜单/权限

几乎所有后台系统都包含“用户管理、角色管理、菜单/权限管理”。原型阶段应把三者的关系表现出来，而不是只做一个孤立的用户列表。

### 用户管理

- **列表字段**：用户ID、头像/昵称、账号、所属部门、角色（多角色用 tag 列表）、手机号、状态（启用/禁用）、最后登录时间、操作。
- **状态切换**：用 `el-switch` 表示启用/禁用，切换时触发 `ElMessageBox.confirm` 确认，文案提示“禁用后该用户将无法登录系统”。
- **重置密码**：行内操作按钮，点击弹出小 Dialog（30%），输入新密码 + 确认密码，带简单校验（长度、两次一致）。
- **分配角色**：新增/编辑用户时，用 `el-select` 多选展示角色；如果角色多，可用 `el-transfer` 展示“已选 / 可选”。
- **分配部门**：用 `el-tree-select` 单选所属部门（树形下拉）。
- **Mock 数据**：用真实中文姓名（如 陈静、王伟、刘洋），部门名如“研发中心 / 财务部”，角色如“系统管理员、运营专员、普通员工”。

### 角色管理

- **列表字段**：角色ID、角色名称、角色编码、数据权限范围、状态、创建时间、操作。
- **数据权限选项**：全部数据、本部门数据、本部门及以下数据、仅本人数据，用 `el-radio-group` 展示。
- **权限分配**：点击“权限分配”打开大 Dialog（70% 或 800px+），左侧/右侧用 `el-tree` 展示菜单权限（树形勾选），支持“展开/折叠/全选/取消全选”。
- **角色编码**：用 `ROLE_ADMIN`、`ROLE_OPERATOR` 等可读编码，不要只写“角色1”。

### 菜单/权限管理

- **页面结构**：左侧菜单树 + 右侧详情/编辑表单。
- **树形展示**：用 `el-tree` 展示菜单层级，图标用 `@element-plus/icons-vue` 对应（如 `Menu`、`User`、`Setting`）。
- **节点字段**：菜单名称、菜单路径（`path`）、组件、图标、权限标识（`system:user:add`）、排序、状态、是否显示。
- **权限标识**：统一写成 `模块:功能:操作` 的形式，如 `system:user:edit`、`order:refund:audit`，不要出现无意义的 `perm_1`。
- **操作**：新增/编辑/删除节点；删除有子节点时，二次确认并提示“删除后将同时删除所有子菜单”。

## 组织架构

- **页面结构**：左侧部门树 + 右侧该部门成员列表（或部门详情）。
- **树操作**：每个节点右侧显示“新增子部门 / 编辑 / 删除”图标按钮，hover 时显示；新增/编辑用中小 Dialog。
- **部门字段**：部门名称、部门编码、负责人、联系电话、排序、状态、创建时间。
- **成员列表**：默认显示当前选中部门的直属成员，加一个“包含子部门”复选框。
- **Mock 数据**：部门树形如“XX科技 / 研发中心 / 前端组 / 后端组”，负责人用真实姓名。
- **删除限制**：删除部门前检查是否包含成员，如果有成员则提示“请先转移该部门成员，再删除部门”。

## 岗位与职级

- **岗位管理**：岗位名称、岗位编码、状态、排序、创建时间。列表CRUD，不需要树。
- **职级管理**：职级名称、职级级别、状态、描述。可用 `el-tag` 区分级别高低（如 P1~P10）。
- **使用场景**：用户编辑时关联岗位和职级，不要做成完全独立的无关模块。

## 数据字典

- **页面结构**：左侧字典类型列表 + 右侧字典项列表。
- **字典类型**：字典名称、字典类型编码（如 `sys_user_status`、`order_priority`）、状态、备注。
- **字典项**：字典标签、字典值、排序、状态、样式（如 `success / info / warning / danger` 对应 tag 颜色）。
- **枚举对应**：业务列表中的状态 tag 颜色要和字典项里定义的样式保持一致（如“启用”用 `success`，“禁用”用 `danger`）。
- **Mock 数据**：
  - 用户状态：`启用/禁用`
  - 订单状态：`待支付/已支付/发货中/已完成/已取消`
  - 优先级：`高/中/低`

## 系统参数/配置

- **页面结构**：参数列表，支持按参数分类筛选（如“基础配置 / 邮件配置 / 文件配置”）。
- **参数字段**：参数名称、参数键名、参数键值、系统内置（是/否）、备注、操作。
- **编辑规则**：
  - 系统内置参数删除时，提示“系统内置参数不可删除”。
  - 参数键值根据类型渲染不同输入：字符串用 `el-input`，布尔用 `el-switch`，数字用 `el-input-number`，枚举用 `el-select`。
- **Mock 数据**：
  - `sys.login.captcha`：是否开启验证码
  - `sys.file.maxSize`：上传文件大小限制（MB）
  - `sys.email.sender`：邮件发件人地址

## 通知公告

- **列表字段**：公告标题、公告类型（通知/公告/提醒）、状态（草稿/已发布/已下线）、发布时间、创建人、操作。
- **状态流转**：草稿 → 发布；已发布 → 下线；下线后可删除。
- **发布操作**：发布按钮用 `type="success"`，点击确认“发布后所有用户可见，是否继续？”。
- **详情展示**：公告详情用富文本区域（原型阶段用 `el-input type="textarea"` 模拟即可），详情 Dialog 展示标题 + 发布时间 + 内容。

## 操作日志与登录日志

这类页面本质上是只读查询列表，不应出现“新增/编辑/删除”按钮，但要体现筛选和详情。

### 操作日志

- **列表字段**：操作模块、操作类型、操作人员、请求方法、请求URL、IP地址、操作结果（成功/失败）、操作时间、耗时（ms）、操作。
- **操作列**：只保留“详情”，点击打开详情 Drawer/Dialog，展示请求参数、响应结果、异常信息。
- **筛选条件**：操作模块（下拉）、操作类型（下拉）、操作人员（输入）、日期范围。
- **Mock 数据**：模块如“用户管理、角色管理、订单管理”，操作如“新增、修改、删除、导出”。

### 登录日志

- **列表字段**：登录账号、登录IP、登录地点、浏览器、操作系统、登录状态（成功/失败）、登录时间、提示信息、操作。
- **操作列**：可加入“强退/踢下线”按钮（如果原系统支持），原型阶段弹出确认即可。
- **筛选条件**：登录账号、登录状态、日期范围。
- **Mock 数据**：IP 用真实格式，如 `192.168.1.100`，浏览器如 `Chrome 120 / Windows 10`。

## 定时任务

- **列表字段**：任务名称、任务组、调用目标（Bean/方法名）、Cron表达式、执行策略（立即执行/错过执行/放弃执行）、任务状态（运行中/暂停）、下次执行时间、创建时间、操作。
- **Cron 编辑**：用 `el-input` 输入 cron 表达式，旁边显示“最近5次执行时间”的只读预览（用 mock 文本）。
- **操作按钮**：
  - 启动/暂停：用 `el-switch` 或按钮切换，切换时确认。
  - 立即执行：点击执行一次，提示“执行成功，请查看执行日志”。
  - 执行日志：行内按钮打开日志列表 Dialog，展示执行结果、耗时、异常信息。
- **Mock 数据**：任务如“每日数据备份（`0 0 2 * * ?`）”、“订单超时取消（`0 */5 * * * ?`）”。

---

## 在线用户

- **列表字段**：会话ID、登录账号、所属部门、登录IP、登录地点、浏览器、操作系统、登录时间、操作。
- **操作列**：只保留“强退”按钮，点击二次确认“强制退出后该用户需重新登录，是否继续？”。
- **实时感**：列表上方显示“当前在线：XX 人”，用统计卡片点缀。
- **删除限制**：在线用户列表不应提供“新增/编辑/删除”功能，只读 + 强退。

---

## 文件管理

- **页面结构**：左侧存储类型/文件夹树 + 右侧文件列表，或单列表加分类筛选。
- **列表字段**：文件名称、文件类型、文件大小、存储路径/访问URL、上传人、上传时间、操作。
- **文件类型**：用图标或 `el-tag` 区分图片、文档、视频、压缩包等。
- **预览**：图片/文档提供“预览”按钮，点击打开 `el-image` 或提示“文件预览”Dialog。
- **操作**：下载、删除。删除确认“删除后无法恢复，是否继续？”。
- **Mock 数据**：文件名如 `2024年度报告.pdf`、`用户头像_001.png`，文件大小用 `KB/MB` 格式化显示。

## 通用交互约定（系统管理类）

1. **状态统一**：启用/禁用、发布/下线、运行/暂停 等状态切换，全部用 `el-switch` 或明确按钮 + 二次确认。
2. **只读页面不暴露写操作**：日志、在线用户等页面不要出现“新增、编辑”按钮，避免视觉干扰。
3. **树形模块操作**：部门、菜单、字典等树形模块，使用“左侧树 + 右侧详情/列表”布局，树的节点操作在 hover 时显示。
4. **分配类操作使用 Dialog**：分配角色、分配权限、分配岗位等，使用 Dialog 或 Drawer，避免跳转路由。
5. **Mock 数据真实**：权限标识用 `system:user:edit`，角色编码用 `ROLE_ADMIN`，cron 用真实表达式，避免 `test`、`abc123`、`role1` 等占位符。
6. **分类/标签筛选**：系统管理模块通常有“分类、类型、状态”筛选，统一放在列表顶部筛选 panel 中，用下拉选择或单选按钮组。
