# 清新浅色风格规范

原型的第一观感靠它。核心气质：干净、透气、柔和——像一张整理过的书桌，不是五彩斑斓的dashboard。

## 色板

按用途分四类（参考 Arco 色彩分类）：**主色**（品牌/强调）、**中性色**（文字/背景/边框）、**功能色**（成功/警告/危险/链接）、**遮罩色**（模态底罩）。配色越克制越清新，不要把功能色当装饰乱用。

| 用途 | 值 | 说明 |
|------|-----|------|
| 页面背景 | `#f6f8fa` | 极浅的灰蓝，比纯白柔和 |
| 卡片/容器 | `#ffffff` | 白色卡片浮在背景上 |
| 主色 primary | `#14b8a6` | 青绿（teal），清新的关键。按钮、选中态、链接强调 |
| 辅助色 | `#0ea5e9` | 浅蓝，图表第二色、信息提示 |
| 成功 | `#22c55e` | |
| 警告 | `#f59e0b` | |
| 危险 | `#ef4444` | |
| 标题文字 | `#1e293b` | 深 slate，不用纯黑 |
| 正文文字 | `#475569` | |
| 次要文字 | `#94a3b8` | 占位、说明、时间戳 |
| 边框/分割线 | `#e2e8f0` | 细而浅 |
| 遮罩色（Dialog/抽屉底罩） | `rgba(15,23,42,0.6)` | 突出模态窗口，配合透明度使用。别用纯黑 `rgba(0,0,0,0.5)`，灰蓝更柔和 |

Element Plus 主题色通过 `theme.css` 里的 `--el-color-primary` 系列 CSS 变量覆盖，已配好 teal 的 hover/active/light 梯度。页面内要强调色时用 `var(--el-color-primary)`，别写死。

## 形状与层次

- **圆角**：卡片 12px，按钮/输入框用 Element Plus 默认（6px 左右），头像、tag 圆形/胶囊
- **阴影**：模拟物理空间关系，要轻。按 Arco 四级阴影体系，层级越高阴影越深：
  - 一级（卡片默认，几乎贴在背景上）：`0 1px 3px rgba(15, 23, 42, 0.06)`
  - 二级（hover 态，轻微浮起）：`0 2px 8px rgba(15, 23, 42, 0.10)`
  - 三级（下拉菜单、popover、select 面板）：`0 4px 12px rgba(15, 23, 42, 0.12)`
  - 四级（Dialog、抽屉、最顶层元素）：`0 8px 24px rgba(15, 23, 42, 0.16)`
  - 禁止浓黑大阴影（如 `0 4px 20px rgba(0,0,0,0.3)`），清新靠"轻"
- **边框**：优先用阴影区分层次，边框只做辅助（表格、分割线），颜色 `#e2e8f0`

## 空间

- 页面内容区 padding 24px；卡片内 padding 20px
- 卡片之间间距 16~20px
- 宁可留白多一点，也不要把内容挤满。清新 = 敢留白

## 字体

系统字体栈即可（theme.css 已配）。无衬线体亲和现代，清晰便于阅读，优先用系统默认无衬线字体。

### 字号层级

字号是信息层级的工具，不是越多越好。一个原型内字号层级保持 4~5 种，层级之间差 2~4px；要突出重点优先加字重，不要靠堆字号。

| 用途 | 字号 | 字重 |
|------|------|------|
| 统计大数字 | 24~32px | 600 |
| 页面标题 | 20px | 600 |
| 卡片标题 | 16px | 600 |
| 正文 | 14px（主字号） | 400 |
| 辅助说明/表头 | 13px | 500 |
| 时间戳/标签/极次要 | 12px（最小） | 400 |

最小可识别字号 12px，别再小。数字（金额、统计值）用 500~600 字重突出。

### 字重

| 字重 | 场景 |
|------|------|
| 400 Regular | 正文、次要文字 |
| 500 Medium | 表头、统计数字、卡片标题、需要轻微突出的文字 |
| 600 Semibold | 页面标题、Dialog 标题、强调标题 |

不要用 700+（Bold/Black），过重不"清新"。

### 行高与段间距

- 正文行高 **1.5**（中文偏密，比 Arco 默认 1.4 稍大，更透气）
- 标题行高 1.3~1.4
- 多行文本段间距至少字号的 1.5 倍（14px 正文 → 段间距 21px+）
- 别让多行文字挤在一起，该空就空

## 组件使用气质

- 统计卡片：大数字 + 小标题 + 可选迷你趋势，icon 用浅色圆形底（`--el-color-primary-light-8` 背景 + primary 色图标）
- 表格：斑马纹或 hover 高亮即可，表头浅灰底（`#f8fafc`），状态用 el-tag 的 light 效果（`effect="light"` 圆角）
- 图表：主色 teal + 浅蓝 + 浅灰系列，grid 线要淡，不要默认的深灰轴线
- 顶栏：白色底 + 底部 1px 浅边框，品牌名用 primary 色，导航选中态 primary 色加底部 2px 指示条
- 插画/图标：Element Plus 自带 @element-plus/icons-vue，够用；不要引入 emoji 当图标用

## 列表页通用结构

管理类列表页统一长这样，避免每个页面各写各的：

1. **顶部筛选 panel**：一个 `el-card`（或同款白底圆角容器）把所有筛选条件收进去。内部 `el-form` + `el-row`/`el-col`，字段按 `xs:24 sm:12 md:8 lg:6` 自适应；日期范围等长字段占 `md:12 lg:8`。查询（primary）/ 重置（默认）按钮放最后一列右对齐。整页就这一个筛选区，不要拆成多块。
2. **表格**：紧跟筛选 panel 下方，同样放一张 `el-card` 里或直接裸表配浅色容器。行点击打开 Dialog，不跳路由。
3. **分页**：表格下方右对齐 `el-pagination`，show total / size changer / jumper 都开。

## Dialog 尺寸约定

行点击 / 编辑 / 新增统一用 `el-dialog`，宽度按内容体量三档：

| 档位 | 宽度 | 适用 |
|------|------|------|
| 小 | `30%` 或 `420px` | 纯文本详情、简单确认、少量字段 |
| 中 | `50%` 或 `600px` | 常规表单、编辑、含少量分组的详情 |
| 大 | `70%` 或 `800px+` | 复杂详情、多 tab、时间线、明细列表 |

标题写明当前对象，如「订单详情：ORD-20240719-001」。详情用 `el-descriptions`，编辑用 `el-form`。

## 删除操作

所有删除（行内删除、批量删除、卡片删除）必须先 `ElMessageBox.confirm`，文案点明"删除后不可恢复"+ 对象名称；确认后本地更新数据 + `ElMessage.success`，取消则什么都不做。删除按钮统一 `type="danger"`，行内可用 `link type="danger"` 文字按钮。
